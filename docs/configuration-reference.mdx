# Configuration Reference

Complete reference for all configuration options in `config.yaml`.

## Organization Settings

Configure your organization's basic information:

| Field | Type | Required | Description |
|-------|------|----------|-------------|
| `org.name` | string | Yes | Organization name |
| `org.description` | string | Yes | Brief description |
| `org.url` | string | Yes | Organization website |
| `org.logo_url` | string | Yes | Logo image URL |
| `org.start_date` | date | No | Organization founding date |

### Social Links

| Field | Type | Required | Description |
|-------|------|----------|-------------|
| `org.socials.github` | string | No | GitHub organization URL |
| `org.socials.slack` | string | No | Slack workspace URL |
| `org.socials.linkedin` | string | No | LinkedIn company URL |
| `org.socials.youtube` | string | No | YouTube channel URL |
| `org.socials.email` | string | No | Support email address |

## Metadata Settings

SEO and site metadata configuration:

| Field | Type | Required | Description |
|-------|------|----------|-------------|
| `meta.title` | string | Yes | Site title |
| `meta.description` | string | Yes | Site description |
| `meta.image_url` | string | Yes | Social media preview image |
| `meta.site_url` | string | Yes | Full site URL |
| `meta.favicon_url` | string | Yes | Favicon URL |

## Leaderboard Settings

Configure leaderboard behavior and data sources:

| Field | Type | Required | Description |
|-------|------|----------|-------------|
| `leaderboard.data_source` | string | Yes | GitHub repository with leaderboard data |
| `leaderboard.theme` | string | No | URL to custom theme CSS |
| `leaderboard.roles` | object | Yes | Role definitions (minimum 1) |
| `leaderboard.top_contributors` | array | No | Activity types to show in top contributors |
| `leaderboard.aggregates` | object | No | Aggregates to display on pages |
| `leaderboard.social_profiles` | object | No | Social profile icon mappings |

### Roles Configuration

Each role in `leaderboard.roles` can have:

| Field | Type | Required | Description |
|-------|------|----------|-------------|
| `name` | string | Yes | Display name for the role |
| `description` | string | No | Description of the role |
| `hidden` | boolean | No | Hide from leaderboard (default: false) |

Example:

```yaml
leaderboard:
  roles:
    core:
      name: Core Team
      description: Core team members
    contributor:
      name: Contributor
      description: Open source contributors
    bot:
      name: Bot
      description: Bot accounts
      hidden: true  # Hide from leaderboard
```

### Aggregates Configuration

Configure which aggregates to display:

| Field | Type | Description |
|-------|------|-------------|
| `leaderboard.aggregates.global` | array | Aggregates for home page |
| `leaderboard.aggregates.contributor` | array | Aggregates for profile pages |

Built-in global aggregates:
- `total_activities` - Total number of activities
- `count_contributors` - Number of contributors
- `activity_types` - Number of activity types

Built-in contributor aggregates:
- `total_points` - Total points earned
- `total_activities` - Total activities
- `activity_types` - Types of activities

## Scraper Configuration

Configure automated data collection (optional):

| Field | Type | Required | Description |
|-------|------|----------|-------------|
| `scraper.schedule` | string | Yes | Cron expression for scheduled execution |
| `scraper.scrapers` | array | Yes | List of scrapers to run |

Each scraper in `scraper.scrapers` has:

| Field | Type | Required | Description |
|-------|------|----------|-------------|
| `name` | string | Yes | Scraper name |
| `repository` | string | Yes | GitHub repository of the scraper |
| `envs` | object | No | Environment variables for the scraper |

Example:

```yaml
scraper:
  schedule: 0 */12 * * *
  scrapers:
    - name: GitHub Scraper
      repository: ohcnetwork/leaderboard-github-scraper
      envs:
        GITHUB_TOKEN: ${{ secrets.GITHUB_TOKEN }}
        GITHUB_ORG: ${{ github.repository_owner }}
    - name: Slack Scraper
      repository: ohcnetwork/leaderboard-slack-scraper
      envs:
        SLACK_API_TOKEN: ${{ secrets.SLACK_API_TOKEN }}
```

## Schema Validation

The configuration is validated against `config.schema.json`. Add this line to the top of your `config.yaml` for IDE support:

```yaml
# yaml-language-server: $schema=config.schema.json
```

This enables:
- Autocomplete in your IDE
- Real-time validation
- Documentation tooltips

## Example Configuration

Complete example with all options:

```yaml
# yaml-language-server: $schema=config.schema.json

org:
  name: Open Healthcare Network
  description: Open-source platform for connected healthcare systems
  url: https://ohc.network
  logo_url: https://ohc.network/logo.svg
  start_date: 2020-03-08
  socials:
    github: https://github.com/ohcnetwork
    slack: https://slack.ohc.network
    linkedin: https://linkedin.com/company/ohcnetwork
    email: support@ohc.network

meta:
  title: Leaderboard - Open Healthcare Network
  description: Track contributions across OHC projects
  image_url: https://ohc.network/og-image.png
  site_url: https://contributors.ohc.network
  favicon_url: https://ohc.network/favicon.ico

leaderboard:
  data_source: https://github.com/ohcnetwork/leaderboard-data
  theme: https://raw.githubusercontent.com/ohcnetwork/leaderboard-data/main/theme.css
  
  roles:
    core:
      name: Core
      description: Core member of OHC
    intern:
      name: Intern
      description: Intern at OHC
    contributor:
      name: Contributor
      description: Open source contributor
    bot:
      name: Bot
      description: Bot account
      hidden: true
  
  top_contributors:
    - pr_merged
    - pr_opened
    - issue_opened
    - issue_assigned
  
  aggregates:
    global:
      - total_activities
      - count_contributors
      - pr_avg_tat
    contributor:
      - total_points
      - total_activities
      - pr_avg_tat
  
  social_profiles:
    github:
      icon: github
    linkedin:
      icon: linkedin
    twitter:
      icon: twitter
    email:
      icon: mail

scraper:
  schedule: 0 */12 * * *
  scrapers:
    - name: GitHub Scraper
      repository: ohcnetwork/leaderboard-github-scraper
      envs:
        GITHUB_TOKEN: ${{ secrets.GITHUB_TOKEN }}
        GITHUB_ORG: ${{ github.repository_owner }}
    - name: Slack Scraper
      repository: ohcnetwork/leaderboard-slack-scraper
      envs:
        SLACK_API_TOKEN: ${{ secrets.SLACK_API_TOKEN }}
        SLACK_CHANNEL: ${{ vars.SLACK_EOD_CHANNEL }}
```
